## Why

用户误传、重复提交或希望撤回展示时，无法在班级列表中移除自己的作品，只能被动保留条目。提供「删除本人作品」可恢复可控的发布体验，并与已存在的上传与作者快照能力形成完整闭环。

## What Changes

- 在班级列表中，对**当前登录用户本人发布**的条目提供**删除**操作（例如每条条目上的删除入口）。
- 删除后该作品从列表与持久化存储中移除，刷新页面后不应再出现；**非作者**不得删除他人条目。
- 删除前提供确认（避免误触）；删除失败时给出明确提示。
- **非目标**：批量删除、管理员代删、回收站/软删除、删除后投票历史的复杂回滚（若后续有投票数据关联，在对应 spec 中另述）。

## Capabilities

### New Capabilities

- （无）行为作为班级作品提交与列表能力的延伸。

### Modified Capabilities

- `class-submissions`: 增加「作者可删除本人已发布作品」的需求与场景（含权限、确认、持久化一致性）。

## Impact

- **前端（`web/`）**：班级列表 UI 增加作者可见的删除控件、确认流程；调用或扩展现有提交/存储层以移除条目及关联媒体。
- **数据**：与当前客户端持久化（如 IndexedDB）一致，需原子或顺序删除元数据与二进制，避免残留孤儿文件。
- **依赖**：依赖已登录态与用户身份与 `uploaderStudentId`（或项目内用于判定「本人」的同等字段）比对逻辑。
