## Why

投票小程序需要区分「谁在操作」，后续上传作品与投票都依赖当前用户身份。在辅助自用、无商业账号体系的前提下，用**姓名 + 学号**作为轻量身份标识即可满足登录与登出需求，并为后续功能提供会话基础。

## What Changes

- 提供**登录**入口：用户必须填写**姓名**与**学号**后才能进入已登录状态。
- 提供**登出**：清除当前会话/本地身份，回到未登录状态。
- 未登录时，与「当前用户」相关的操作（若已存在页面）应引导先登录或隐藏；具体与上传/投票的衔接在后续变更中扩展。

## Capabilities

### New Capabilities

- `user-login`: 基于姓名与学号的登录、登出及当前用户会话（或等价本地身份）的展示与校验规则。

### Modified Capabilities

- （无）当前仓库尚无 `openspec/specs/` 下的既有能力说明。

## Impact

- **前端**：登录/登出 UI、表单校验、登录态在路由或布局中的体现（具体技术栈在实现阶段确定）。
- **存储**：至少需在客户端持久化「当前用户」信息（如 localStorage / sessionStorage / Cookie 等，由设计文档定稿）；若后续接后端，可迁移为服务端会话。
- **依赖**：无强制新增外部服务；若未来接入真实鉴权，本能力的行为可作为迁移基线。
