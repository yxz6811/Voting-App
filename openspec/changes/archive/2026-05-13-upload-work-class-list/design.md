## Context

投票小程序 `web/` 已具备基于 `useAuth` 的本地登录态（见 `openspec/specs/user-login/spec.md`）。本变更在**无服务端**前提下，让用户把**图片或视频**作品提交到**班级列表**并在同页或独立视图中浏览；后续投票与多端同步不在本变更范围。

## Goals / Non-Goals

**Goals:**

- 已登录用户可选择本地文件并提交；提交后**立刻**在班级列表中出现一条可识别作者与媒体类型的记录。
- 在浏览器内**持久化**作品二进制与元数据，使刷新页面后列表仍可恢复（直至用户清除站点数据）。
- 图片与视频均具备**可理解的预览**（图片内联预览；视频使用 `<video>` 播放本地对象 URL 或等价机制）。

**Non-Goals:**

- 云端上传、CDN、转码、封面自动生成（可后续增强）。
- 作品审核流、版权检测、病毒扫描。
- 班级成员「权限模型」与学号唯一性约束（仍沿用当前自用场景假设）。

## Decisions

1. **二进制存储使用 IndexedDB**  
   - **选择**：以 `submissionId`（UUID 或等效唯一字符串）为主键，将 `Blob` 存入 IndexedDB；元数据（`submissionId`、`uploaderDisplayName`、`uploaderStudentId`、`mimeType`、`originalFileName`、`byteSize`、`createdAt`）与主键一并存入同一 object store 或拆分为 `meta` + `blob` 两 store。  
   - **理由**：`localStorage` 不适合大视频；IndexedDB 配额更适合自用演示与中小体积作品。  
   - **备选**：纯内存 + `sessionStorage` 元数据 — 刷新即丢，与「进入班级列表」的期望不符。

2. **作者信息在提交时快照**  
   - **选择**：写入提交记录时使用当前 `useAuth().user` 的姓名与学号作为快照字段，避免用户登出/换号后历史条目作者语义混乱。  
   - **理由**：班级列表展示的是「提交当时」的贡献者。  
   - **备选**：仅存 `studentId` 并每次渲染查当前 profile — 实现复杂且无后端时收益有限。

3. **媒体类型白名单（实现与 spec 对齐）**  
   - **图片**：`image/jpeg`、`image/png`、`image/webp`、`image/gif`。  
   - **视频**：`video/mp4`、`video/webm`（MIME 以浏览器 `File.type` 为准；若为空则辅以扩展名启发式，**设计建议**：优先信任 `type`，缺失时再回退扩展名并在 spec 中写清 MUST 行为）。  
   - **理由**：覆盖常见网页可预览格式，降低无法播放的概率。

4. **单文件大小上限**  
   - **选择**：**80 MiB**（`80 * 1024 * 1024` 字节）为硬上限；超出 MUST 拒绝提交并给出可读错误。  
   - **理由**：在 IndexedDB 与自用场景之间折中；数值写入 spec 以便测试。  
   - **备选**：100 MiB — 可按 Open Questions 调整。

5. **列表与上传的 UI 组织**  
   - **选择**：在已登录壳层内增加主导航或分区：「上传作品」「班级列表」；列表按 `createdAt` **新到旧**排序。  
   - **理由**：符合心智模型，便于后续接投票入口。

6. **预览释放**  
   - **选择**：对通过 `URL.createObjectURL` 创建的 URL 在删除条目或页面卸载路径上 `revokeObjectURL`，避免内存泄漏。  
   - **理由**：列表反复操作时的稳定性。

## Risks / Trade-offs

- **[风险] 浏览器存储配额不足** → **缓解**：捕获写入错误并提示用户删除旧作品或缩小文件；可在后续变更增加「删除我的作品」。  
- **[风险] 部分视频编码在 Safari/移动端不可播** → **缓解**：文档化推荐 H.264+AAC 的 MP4；不可播时展示明确错误占位（仍保留条目）。  
- **[权衡] 无服务端 = 数据仅本机** → **缓解**：在 UI 文案中弱提示（非阻断），未来接 API 时迁移。

## Migration Plan

- 首版 IndexedDB schema 带版本号；若后续字段变更，使用 `indexedDB.open` 升级回调做迁移或清空旧版本（自用可接受重建）。  
- 若未来接后端：可增加「同步到班级」步骤，本阶段数据结构尽量与「提交记录」概念一一对应，便于映射。

## Open Questions

- 是否需要**作品标题**独立字段（当前可默认用原始文件名展示）。  
- 是否需要在首版提供**删除已提交作品**（未在提案中强制，可单开变更）。
