## Why

登录已能标识「谁在上传」，但投票小程序的核心价值之一是让每位同学把**自己的作品**放进**班级列表**供浏览与后续投票。当前缺少从本地上传**视频或图片**并进入列表的能力；在网页端先打通该闭环，才能与 `RESEARCH.md` 中的上传与预览目标对齐。

## What Changes

- 仅在**已登录**状态下提供「上传作品」入口；未登录时须引导登录（与既有 `user-login` 行为衔接，不在本变更中改写登录规则）。
- 支持用户从设备选择**图片**和/或**视频**文件作为作品提交到**班级列表**（列表中展示条目，含作者身份与可预览的媒体类型）。
- 提供**班级列表**视图：展示已提交的作品条目（至少含作者展示名、媒体类型、可访问的预览或占位）；具体排序与投票逻辑由后续变更扩展。
- **非目标**：跨设备云端同步、多人实时协作编辑、服务端转码与版权检测（若未来接入后端，可在后续变更中替换存储实现）。

## Capabilities

### New Capabilities

- `class-submissions`: 已登录用户将图片或视频作品提交到班级列表，并在列表中展示与预览（客户端持久化策略见 `design.md`）。

### Modified Capabilities

- （无）不改变 `openspec/specs/user-login/spec.md` 中的登录/登出需求文本；本变更在上层功能中依赖已登录态。

## Impact

- **前端（`web/`）**：新增上传流程、班级列表 UI、与 `useAuth` 的衔接；需引入或扩展客户端存储以承载媒体二进制与列表元数据。
- **数据**：默认无服务端；存储形态（如 IndexedDB + 元数据索引）在 `design.md` 定稿，可能影响包体与浏览器配额策略。
- **依赖**：可能增加与文件读取、缩略图/预览相关的浏览器 API 使用；不强制新增 npm 依赖，除非设计阶段选定具体库。
